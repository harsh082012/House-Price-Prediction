## 📖 Overview

This project estimates Bangalore home prices using a regression model trained on the [Bengaluru House Price dataset](model/Bengaluru_House_Data.csv.xls). The trained model is served through a FastAPI backend, and a simple client lets users pick a location, enter square footage, BHK, and bathrooms to get an instant price estimate.

| Layer | Technology |
|---|---|
| Model training | Python, pandas, scikit-learn (Jupyter notebook) |
| Backend API | FastAPI, Uvicorn |
| Frontend | HTML, CSS, vanilla JavaScript |
| Deployment | Render |

## ⚙️ How It Works

1. **Training** — `model/housing.ipynb` cleans the raw dataset, engineers features (price per sqft, location grouping, outlier removal), and trains a linear regression model.
2. **Serialization** — The trained model is saved as `banglore_home_prices_model.pickle`, alongside `columns.json` which records the exact feature/location columns the model expects.
3. **Serving** — `server/util.py` loads these artifacts once at startup. `server/server.py` exposes them as two FastAPI endpoints.
4. **Prediction** — The client sends the chosen location, square footage, BHK, and bathrooms to the API and displays the returned estimate.

## 🔌 API Reference

Base URL (local): `http://127.0.0.1:8000`
Base URL (production): `https://house-price-prediction-q0vx.onrender.com`

### Get all available locations

```http
GET /get_location_names
```

**Response**

```json
{
  "locations": ["1st phase jp nagar", "whitefield", "indira nagar", "..."]
}
```

### Predict a home price

```http
POST /predict_home_price
Content-Type: application/json
```

**Request body**

```json
{
  "total_sqft": 1000,
  "location": "1st phase jp nagar",
  "bhk": 3,
  "bath": 3
}
```

**Response**

```json
{
  "estimated_price": 90.92
}
```

> Prices are returned in lakhs (₹ 1 lakh = ₹100,000), matching the dataset's original unit.

Interactive API docs (Swagger UI) are auto-generated by FastAPI and available at `/docs` on whichever URL you're running — try every endpoint directly from your browser without writing any code.

## 🚀 Getting Started Locally

### Prerequisites

- Python 3.10+
- pip

### 1. Clone the repository

```bash
git clone https://github.com/harsh082012/House-Price-Prediction.git
cd House-Price-Prediction
```

### 2. Set up the backend

```bash
cd server
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn server:app --reload
```

The API will be running at `http://127.0.0.1:8000`. Visit `http://127.0.0.1:8000/docs` to test it interactively.

### 3. Run the client

Open `client/app.html` directly in your browser, or serve it locally:

```bash
cd ../client
python -m http.server 5500
```

Then visit `http://127.0.0.1:5500/app.html`.

> Make sure `app.js` points to your running API URL (`http://127.0.0.1:8000` locally, or your Render URL in production).

## ☁️ Deployment

This project is deployed on [Render](https://render.com):

| Setting | Value |
|---|---|
| Root Directory | `server` |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `uvicorn server:app --host 0.0.0.0 --port $PORT` |
| Runtime | Python 3 |

The client is deployed as a separate Render Static Site, with `app.js` pointed at the live API URL above. CORS on the backend is restricted to this client's origin.

> ⏱️ **Note:** This app runs on Render's free tier, which spins down after periods of inactivity. The first request after idle time may take 30–60 seconds to respond while the server wakes up.
